# Healthcare AI Intake + Prioritization & Value Framework

> A structured operating model for evaluating, prioritizing, and measuring the value of AI and automation investments in healthcare.

---

## Purpose

This repository provides a comprehensive, reusable framework for healthcare organizations to:

1. **Intake and evaluate** AI/automation use case submissions in a consistent, structured way
2. **Prioritize** initiatives using a transparent, weighted scoring model
3. **Define and measure value** across financial, clinical, operational, and experiential dimensions
4. **Track KPIs** from initiative launch through sustained business impact

This framework is designed to be adopted by health systems, payers, and healthcare services organizations looking to build a disciplined approach to AI portfolio management.

## How to Navigate This Framework

### Step 1 — Submit a Use Case
Use the **[Intake Form Template](templates/intake-form-template.md)** to capture all relevant details about a proposed AI or automation initiative. This ensures consistent information across all submissions.

Reference **[Intake & Prioritization Framework](docs/intake-prioritization-framework.md)** for submission guidelines and the end-to-end process.

### Step 2 — Score and Prioritize
The evaluation committee uses the **[Prioritization Scorecard Template](templates/prioritization-scorecard-template.md)** to apply the weighted scoring model described in **[Scoring Model](docs/scoring-model.md)**.

### Step 3 — Define Value and KPIs
Before launch, each initiative must define its expected value using the **[Value Framework](docs/value-framework.md)** and select relevant KPIs from the **[KPI Library](docs/kpi-library.md)**.

### Step 4 — Track and Report
Use the **[KPI Dashboard Template](templates/kpi-dashboard-template.md)** to monitor performance against baseline and targets throughout the initiative lifecycle.

### Step 5 — Govern and Learn
Follow the **[Governance Model](docs/governance.md)** for review cadence, escalation paths, and portfolio-level reporting. Reference the **[Implementation Guide](docs/implementation-guide.md)** for rollout guidance.

## Value Framework at a Glance

This framework measures value across six domains:

| Domain | Description |
|---|---|
| **Quality & Speed** | Clinical quality, diagnostic accuracy, process throughput, and cycle time reduction |
| **Cost Savings** | Labor efficiency, supply chain optimization, and operational cost avoidance |
| **Patient Experience** | Access, satisfaction, communication quality, and care continuity |
| **Revenue Uplift** | Denial reduction, coding accuracy, capacity optimization, and new revenue streams |
| **Workforce Experience** | Staff satisfaction, burnout reduction, administrative burden, and retention |
| **Risk & Resiliency** | Compliance, audit readiness, safety events, and operational continuity |

## Scoring Model Summary

Initiatives are scored on a **0–100 weighted composite scale** across five dimensions:

| Dimension | Weight |
|---|---|
| Strategic Alignment | 25% |
| Value Potential | 30% |
| Feasibility & Readiness | 20% |
| Risk | 15% |
| Urgency | 10% |

See **[Scoring Model](docs/scoring-model.md)** for full rubrics and tier definitions.

## Getting Started

To adopt this framework in your organization:

1. Read the **[Implementation Guide](docs/implementation-guide.md)**
2. Customize the intake form and scoring weights to fit your organization's priorities
3. Establish a governance committee using the model in **[Governance](docs/governance.md)**
4. Pilot with 3–5 active use cases before full rollout
